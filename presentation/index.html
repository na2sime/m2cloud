L'enquête : un 504 qui n'existait pas en local

01pods Ready, test interne web:80 → 200 en 8 ms ✓
02logs ingress : « upstream timed out »
03matrice de joignabilité : gw:3000 ✓ · web:80 ✗ — sur les deux nœuds
Le SG des nœuds n'autorise le cross-node que sur 1025-65535. Le web écoute sur 80 → bloqué.
Fix : règle SG port 80 + topologySpread + le smoke teste désormais / aussi.
m2cloud@eks:~/rex $ lessons --learned

Ce que j'en retiens

  • Le « ça marche en local » ne suffit pas : RDS, EKS, les registries ont des comportements propres au cloud.
  • On débugge par isolation (interne vs externe, matrice de joignabilité), pas à l'intuition.
  • Il faut tester ce qu'on déploie vraiment : un smoke partiel donne une fausse confiance.
  • Chaque incident → une ligne d'IaC versionnée + un test. Jamais un patch manuel oublié.
